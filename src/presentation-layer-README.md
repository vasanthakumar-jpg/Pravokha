# Presentation Layer

This layer contains pure UI components and layouts. Logic should be kept to a minimum, primarily handling presentation-specific state.

## Structure
- `/ui`: Atomic UI components (primitives), primarily generated by Shadcn/UI.
- `/layout`: Page-level layouts that wrap main content (e.g., `AdminLayout`, `SellerLayout`, `MainLayout`).

## Principles
- **Pure UI**: Components here should not know about business domain logic.
- **Atomic**: Keep components small and focused.
- **Theme-Aware**: All components must use the project's styling and theme tokens.
